What is Challenge Passage?

Challenge Passage is a configurable setting that specifies the length of time that a visitor, who has successfully completed a Challenge (Javascript Challenge, Interactive Challenge or Managed Challenge), can access your website. When the configured timeout expires, the visitor will be issued a new challenge. Challenge Passage does not apply to Rate Limiting.

Challenge Types

Interactive Challenge

This type of challenge offers the highest security by requiring a visitor to solve an interactive challenge on a specific page.

Managed Challenge (Recommended)

Cloudflare's recommended challenge dynamically selects the appropriate challenge type for a visitor based on their browser's signals. Most human visitors are automatically verified, but if Cloudflare detects non-human attributes, an interactive challenge may be presented.

JavaScript Challenge (non-interactive)

A JavaScript (JS) Challenge determines if a visitor is automated based on their browser's signals. The visitor must wait for their browser to process the injected JavaScript, which typically takes less than five seconds. If the challenge passes, the original request is sent to the destination URL. If it fails, another challenge page is presented.
